GreenEnergy Insights: Un Portal Inteligente para la Energía [cite: 3, 4]

Resumen del Proyecto

[cite_start]"GreenEnergy Insights" es una plataforma de análisis predictivo diseñada para abordar el desafío del creciente consumo energético en los Centros de Procesamiento de Datos (CPDs)[cite: 19, 21]. [cite_start]El objetivo principal es evolucionar de una gestión energética reactiva a una estrategia predictiva [cite: 23][cite_start], permitiendo anticipar fluctuaciones en la red eléctrica y responder a ellas antes de que impacten la operación[cite: 24].

Arquitectura Técnica

El proyecto se ha implementado siguiendo una arquitectura de microservicios, donde cada componente es responsable de una tarea específica:

  • [cite_start]Ingesta de Datos (ETL): Un proceso automatizado extrae continuamente datos de la API de Red Eléctrica de España (ESIOS)[cite: 53]. [cite_start]Estos datos son transformados y cargados en una base de datos Azure SQL[cite: 61, 78], utilizando un sistema de tablas "caliente" (para datos recientes) y "fría" (para el histórico).
  • [cite_start]Motor de Analítica (IA): Un script de entrenamiento utiliza los datos históricos para entrenar un modelo de Deep Learning (LSTM)[cite: 34, 54]. [cite_start]Este modelo es capaz de predecir la demanda energética a corto plazo y detectar anomalías[cite: 55]. El modelo se versiona y se "promociona" automáticamente a producción si sus métricas (RMSE, MAPE) superan a la versión anterior.
  • [cite_start]Portal de Visualización (Esta web): Esta aplicación (desarrollada en Flask) carga el modelo promocionado y proporciona una interfaz web interactiva[cite: 38]. [cite_start]Permite a los usuarios generar predicciones futuras, comparar predicciones pasadas con datos reales y analizar las anomalías detectadas[cite: 56].
  • [cite_start]API de Datos (FastAPI): Un servicio independiente de alto rendimiento expone una API REST [cite: 58] que permite a otros sistemas (como el Dashboard o el Chatbot) consultar los datos históricos de la base de datos de forma eficiente.
  • [cite_start]Dashboard y Asistente: El proyecto se complementa con un dashboard de KPIs (Power BI) [cite: 39] [cite_start]y un futuro asistente conversacional (Chatbot) [cite: 41, 59] para consultar las predicciones en lenguaje natural.
